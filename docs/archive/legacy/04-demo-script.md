# 04 演示脚本

## 演示目标

这份脚本用于毕业答辩、项目演示和简历项目讲解。

目标不是展示所有细节，而是用最短路径讲清楚：

1. 世界线是什么
2. 为什么不是从零开发
3. 平台层和模块层如何分离
4. PoE 模块如何形成推荐、图谱和 Agent 的联动闭环

## 演示时长建议

- 标准版：6 到 8 分钟
- 压缩版：3 到 5 分钟

## 演示顺序

### 第一步：首页

打开：

`http://localhost:5173/`

讲解重点：

- 世界线是多主题兴趣知识平台
- 项目不是从零开发，而是基于成熟底座进行产品化改造
- 首期只完整落地 PoE 模块

### 第二步：主题分区

打开：

`http://localhost:5173/themes`

讲解重点：

- 世界线的平台层负责统一承接主题入口
- 当前只完整落地 PoE
- 后续可以扩展其他兴趣主题

### 第三步：PoE 主题页

打开：

`http://localhost:5173/themes/poe`

讲解重点：

- 这里是模块层入口
- 当前已经接入推荐候选与图谱闭环
- 推荐和图谱不是孤立页面，而是平台层可承接的模块上下文

### 第四步：推荐候选 -> Agent

在 PoE 主题页选择一条推荐候选，进入 Agent 对话页。

讲解重点：

- 平台把当前推荐写入主题上下文
- Agent 页能够识别当前推荐与当前闭环
- 说明平台层和模块层不是写死耦合，而是通过上下文承接

### 第五步：图谱闭环 -> Graph

在 PoE 主题页选择一条图谱闭环，进入 Graph 页。

讲解重点：

- Graph 页最小读取当前闭环上下文
- 当前实现重点是演示“闭环入口 + 上下文承接”，不是完整图数据库产品化

### 第六步：后台与底座能力

打开：

- `http://localhost:5173/dashboard`
- `http://localhost:5173/database`

讲解重点：

- 项目不是只做了一个聊天页
- 原有知识库、图谱、后台能力仍被保留
- 世界线是在底座之上做平台化改造

## 答辩时的标准叙述

建议你用下面这段话概括：

> 我的项目名称是“世界线”。它不是从零开发，而是基于 Yuxi-Know 做二次开发与产品化改造。我保留了原有的知识库、RAG、知识图谱、智能体与后台能力，在此基础上构建了世界线的平台层，并以《流放之路》作为首期完整主题模块，验证了推荐候选、知识卡、图谱闭环和 Agent 对话之间的联动能力。

## 如果老师问“你到底做了什么”

建议回答：

- 我不是替换了底座，而是重构了平台层。
- 我完成了世界线品牌、首页、主题分区、PoE 主题页和主题上下文机制。
- 我设计并落地了 PoE 首批结构化知识资产、图谱闭环样本和推荐候选样本。
- 我把这些模块能力通过平台层接入到了 Agent 和 Graph 页面。

## 演示前检查清单

1. Docker 服务已启动
2. 前端可访问
3. 登录账号可用
4. 首页、PoE 主题页、Agent 页、Graph 页都可打开
5. 演示用推荐候选与图谱闭环已显示
