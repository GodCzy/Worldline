# 07 完整使用教程

## 这份文档是给谁看的

这份教程是给“不看代码也要会用项目”的人准备的。

如果你是以下任意一种情况，这份文档就是给你用的：

- 想知道世界线现在到底能做什么
- 想自己本地启动并演示
- 想看懂页面上的每个区域是什么意思
- 想知道从哪个页面进入、按什么顺序点击
- 想准备毕业答辩，但不想靠猜界面功能

## 一、世界线现在是什么

世界线是一个多主题兴趣知识平台。

当前第一期只完整落地了一个主题模块：

- 《流放之路》PoE

它不是一个纯聊天机器人，也不是一个单纯的知识库后台，而是把以下几类能力放到了同一个平台里：

- 平台首页与主题分区
- PoE 主题页
- 推荐候选入口
- 图谱闭环入口
- Agent 对话页
- Graph 图谱页
- Dashboard / 知识库 / 扩展后台

## 二、你现在能用它做什么

### 1. 平台展示

你可以展示：

- 世界线首页
- 主题分区页
- PoE 主题模块页

### 2. PoE 模块演示

你可以展示：

- 推荐候选
- 图谱闭环
- 样本知识卡逻辑
- PoE 上下文如何被带到对话页

### 3. Agent 对话演示

你可以：

- 进入智能体对话页
- 在 PoE 推荐上下文下继续提问
- 展示上下文卡片如何影响当前对话语境

### 4. 图谱演示

你可以：

- 进入图谱页
- 展示当前闭环标题
- 展示 Graph 页面最小承接主题上下文的能力

### 5. 后台底座演示

你可以展示：

- Dashboard
- 知识库页
- 图谱页
- 扩展页

这部分用于说明：你的项目不是只做了“一个前端页面”，而是保留了底座能力。

## 三、从零开始的使用顺序

如果你第一次打开世界线，建议按下面顺序使用。

### 步骤 1：打开首页

地址：

`http://localhost:5173/`

你会看到世界线的首页。

这里的作用是告诉别人：

- 这是什么平台
- 当前首期主题是什么
- 平台和模块是怎么划分的

### 步骤 2：进入主题分区

点击导航里的：

- `主题分区`

或者直接打开：

`http://localhost:5173/themes`

这里会显示当前已经接入的平台主题。

目前重点就是：

- `流放之路 / Path of Exile`

### 步骤 3：进入 PoE 主题页

打开：

`http://localhost:5173/themes/poe`

这个页面是你后续展示 PoE 模块的核心入口。

你会在这里看到三类内容：

1. 当前主题信息
2. 推荐候选
3. 图谱闭环入口

### 步骤 4：选择推荐候选

在 PoE 主题页中，你会看到若干推荐候选卡片，例如：

- SRS 死灵师开荒候选
- 锐眼刷图候选
- 碎骨近战推进候选

每张卡一般会有两个动作：

- 写入推荐上下文
- 带此推荐去对话

如果你只是想先“切换当前推荐”，用：

- `写入推荐上下文`

如果你想直接进入聊天演示，用：

- `带此推荐去对话`

### 步骤 5：进入 Agent 对话页

进入 Agent 页后，页面上方会出现几个和当前主题相关的说明区。

这些区域不是聊天记录，它们是“当前对话场景说明”。

一般包括：

- 当前主题上下文
- 当前推荐
- 当前闭环
- 推荐欢迎提示
- 建议提问按钮

然后页面更下方才是真正的聊天区域。

### 步骤 6：输入问题并继续对话

真正的输入框在页面最底部。

如果页面内容很多，你需要往下看，或者直接滚动到底部。

输入框的作用是：

- 手动输入你的问题
- 继续当前主题上下文下的对话

如果当前已经进入某个推荐候选场景，你可以直接问：

- 这个推荐为什么适合我？
- 我下一步该先补什么？
- 这个 Build 的核心优势是什么？

### 步骤 7：进入 Graph 图谱页

你也可以在 PoE 主题页上选择某条图谱闭环。

如果你是管理员，通常会进入：

- `/graph`

这个页面目前的作用是：

- 展示当前闭环标题
- 承接当前图谱上下文
- 作为图谱能力的最小展示入口

## 四、你给的这张图到底怎么看

你给的截图其实是“Agent 对话页”的一个完整状态。

我按区域解释：

### 1. 左侧窄图标栏

从上到下通常是这些入口：

- 首页
- 主题分区
- Agent
- Graph
- 知识库
- 扩展/后台相关

作用：

- 快速切换模块页面

### 2. 左侧会话栏

这里显示的是：

- 当前智能体下的历史对话列表
- `创建新对话` 按钮

作用：

- 管理聊天会话
- 切换不同历史对话

### 3. 顶部横幅：主题上下文

你截图最上面那一行：

- `主题上下文`
- `poe / poe / SRS 死灵师开荒候选`
- 后面一长串 `scene=... version=... candidate=...`

这部分不是报错，也不是系统日志。

它表示：

- 你当前不在“普通聊天”
- 你是在一个带 PoE 场景的上下文里聊天
- 当前选中的推荐候选是 `SRS 死灵师开荒候选`

简单理解：

它告诉你“你现在是带着什么背景在问问题”。

### 4. 当前推荐卡片

你截图左上大卡片：

- `当前推荐`
- `SRS 死灵师开荒候选`

作用：

- 告诉你现在系统认为你正在围绕哪条推荐方向继续问

### 5. 当前闭环卡片

你截图右上大卡片：

- `当前闭环`
- `死灵师召唤闭环`

作用：

- 告诉你当前推荐对应的是哪条图谱闭环

### 6. 推荐欢迎提示

你截图中间偏左大块说明：

- `欢迎沿着 “SRS 死灵师开荒候选” 继续提问`

作用：

- 告诉你当前最适合问什么方向的问题

### 7. 右侧几个蓝边按钮

这些不是系统按钮，而是“建议提问”。

作用：

- 你可以直接点击其中一个问题，作为快捷提问入口

适合演示时用，因为：

- 不容易卡住
- 容易把对话引导到当前推荐主题

### 8. 中间聊天记录区

你截图中间的：

- `你好世界线`
- 下方工具调用结果

这是已经发送过的一条聊天消息。

作用：

- 显示你之前问过的问题和系统返回过程

### 9. 你说“没对话框了”的地方

其实输入框还在，只是它在页面的最底部。

你截图最下面能看到：

- `输入问题...`

这就是聊天输入框。

如果你觉得“选了候选之后看不到输入框”，通常原因有两个：

1. 页面上方新增了上下文说明区，把聊天区往下推了
2. 你视线停留在中上部，没有滚到最底部

所以你现在该做的是：

1. 先看最底部
2. 找到 `输入问题...`
3. 在那里输入问题

## 五、如果我要从这张图继续操作，具体该怎么做

假设你现在就在这张图这个页面，下一步这样做：

### 情况 A：你想继续聊天

1. 把页面视线移到最底部
2. 找到输入框：`输入问题...`
3. 输入一个问题，例如：
   - 这套 SRS 开荒为什么适合新手？
   - 我下一步先补防御还是先补伤害？
4. 点击右下角发送按钮

### 情况 B：你想开一个新对话

1. 点击左侧会话栏里的 `创建新对话`
2. 新建后会出现新的会话
3. 再在底部输入框中发起问题

### 情况 C：你想切换别的 PoE 推荐

1. 回到 `主题分区 -> PoE 主题页`
2. 换一条推荐候选
3. 再点 `带此推荐去对话`

### 情况 D：你想看图谱

1. 回到 PoE 主题页
2. 选择一条图谱闭环
3. 进入 `/graph`

## 六、完整功能使用说明

### 首页

用途：

- 项目入口
- 品牌展示
- 平台定位说明

### 主题分区页

用途：

- 展示当前有哪些主题模块
- 进入 PoE 模块

### PoE 主题页

用途：

- 展示主题说明
- 展示推荐候选
- 展示图谱闭环

### Agent 页

用途：

- 进行对话
- 展示主题上下文
- 承接推荐与图谱选择结果

### Graph 页

用途：

- 展示图谱相关页面
- 呈现当前闭环信息
- 作为图谱能力演示页

### Dashboard

用途：

- 展示后台管理能力
- 用于说明底座能力保留

### 知识库页

用途：

- 展示知识库管理能力
- 说明世界线不是只做了前台页面

## 七、如果你是答辩展示，推荐怎么点

推荐顺序：

1. 首页
2. 主题分区
3. PoE 主题页
4. 选择推荐候选
5. 进入 Agent
6. 发一个问题
7. 返回 PoE 主题页
8. 选择图谱闭环
9. 进入 Graph
10. 最后展示 Dashboard

## 八、你现在最容易误解的点

### 误解 1：顶部上下文说明就是聊天框

不是。

顶部是“当前场景说明”，真正输入框在底部。

### 误解 2：建议提问按钮就是历史消息

不是。

它们是快捷提问入口，你点了才会发起问题。

### 误解 3：选了推荐以后就没法继续输入了

不是。

只是页面上方多了上下文卡片，输入框被挤到了更下方。

### 误解 4：Graph 页和 Agent 页是两个完全没关系的页面

不是。

它们通过 `themeContext` 承接同一条推荐或闭环语境。

## 九、如果你只想记最关键的三句话

1. 世界线现在已经能跑通“PoE 推荐 -> Agent -> Graph”的演示链路。
2. 你截图里上面那些大卡片是场景说明，不是输入区。
3. 真正聊天输入框永远看页面最底部的 `输入问题...`。
