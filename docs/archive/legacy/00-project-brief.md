# 00 项目说明

## 项目名称

世界线（Worldline）

## 项目定义

世界线是一个基于 Yuxi-Know 进行二次开发与产品化改造的多主题兴趣知识平台。项目目标不是重写底座，而是在保留现有知识库、RAG、知识图谱、智能体和后台配置能力的前提下，完成一套更适合兴趣内容场景的产品层。

首期完整落地主题为《流放之路》（Path of Exile，PoE）。

## 项目背景

兴趣内容领域常见问题包括：

- 内容知识点分散，用户缺乏统一入口
- 内容关系复杂，仅靠关键词检索表达不足
- 用户问题往往带有明确场景，但通用问答缺乏主题上下文承接

PoE 具有复杂职业体系、升华体系、技能搭配、装备成长与机制联动，天然适合作为“知识分类 + 图谱 + 推荐 + Agent 对话”联合验证的首期主题模块。

## 项目目标

本项目目标分为四类：

### 产品目标

- 形成世界线平台品牌与统一入口
- 建立多主题扩展能力
- 完成 PoE 首期主题模块闭环

### 技术目标

- 保留原有底座能力
- 平台层与模块层分离
- 优先用最小改动实现产品化能力

### 交付目标

- 可本地运行
- 可现场演示
- 可用于毕业答辩
- 可写入简历

### 文档目标

- 形成正式项目说明
- 形成架构、路线图、演示脚本和论文材料
- 形成可持续维护的仓库说明体系

## 项目边界

当前阶段明确不做的内容：

- 不进行底座级大规模重构
- 不把 PoE 逻辑硬编码进全局核心模块
- 不把样本数据误包装成完整商业级内容系统

## 当前成果概述

当前项目已经具备以下可交付成果：

- 世界线品牌、首页、主题分区、PoE 主题页
- PoE 推荐候选、知识卡样本、图谱闭环样本
- Agent 与 Graph 页面中的主题上下文联动
- 本地 Docker 运行链路
- 答辩脚本、论文材料与文档站

## 项目价值

这个项目的价值不在于“重新发明一个知识平台”，而在于展示：

1. 如何在成熟底座上进行产品化重构
2. 如何把平台层与模块层清晰分离
3. 如何用一个高复杂度主题模块验证整个平台方案

这也是毕业设计答辩最容易形成清晰叙事的方向。
