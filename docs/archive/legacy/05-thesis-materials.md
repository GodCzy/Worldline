## 可强调的创新点

建议强调以下几点，而不是说“我做了很多功能”：

1. 基于成熟底座进行产品化改造，而非简单套壳
2. 明确区分平台层与模块层
3. 用 PoE 首期模块验证推荐、图谱与 Agent 上下文联动
4. 项目兼具运行能力、展示能力与答辩可讲述性

## 高概率提问与答法

### 为什么不从零开发？

答：

从零开发会把大量时间消耗在底座建设上，不利于毕业设计聚焦核心问题。我的重点是平台层与主题模块如何建立在成熟底座之上，并形成有产品表达的系统。

### 为什么选择 PoE？

答：

PoE 具有复杂职业、技能、装备和机制体系，既适合知识分类，也适合图谱和推荐闭环验证，是一个高复杂度但边界清晰的主题模块。

### 你真正新增的工作是什么？

答：

我新增的重点不是底座能力本身，而是世界线平台层、PoE 模块结构方案、样本知识资产、推荐候选、图谱闭环以及主题上下文在页面之间的承接机制。

### 目前还有什么不足？

答：

样本资产还没有完全接入正式知识导入链，推荐仍是 MVP 级规则方案，图谱也主要是展示型闭环，未来可以继续深化。
