# 03 实施路线图

## 路线图目标

这份路线图用于说明世界线不是无序堆功能，而是按“平台优先、主题次之、低风险落地”的方式推进。

## 已完成阶段

### 阶段 1：仓库映射与规则建立

已完成内容：

- 仓库结构映射
- 入口点识别
- 平台层 / 模块层 / 高风险层边界划分
- 多代理规则与仓库协作规则建立

### 阶段 2：平台层最小实现

已完成内容：

- 世界线品牌入口
- 首页与主题分区
- `/themes/poe`
- 主题上下文 store
- `/agent` 与 `/graph` 的低风险上下文承接

### 阶段 3：PoE 模块方案设计

已完成内容：

- 知识分类
- metadata schema
- 问答模板设计
- 图谱节点与关系设计
- 推荐 MVP 方案设计

### 阶段 4：PoE 样本知识资产

已完成内容：

- 13 张样本知识卡
- manifest 索引
- 样本字段对齐 metadata schema

### 阶段 5：图谱与推荐样本

已完成内容：

- 3 条图谱闭环
- 3 条推荐候选

### 阶段 6-8：平台层最小接入与收口

已完成内容：

- `/themes/poe` 展示推荐与图谱入口
- 推荐候选与图谱选择写入 `themeContext`
- `/agent` 展示当前推荐与闭环信息
- `/graph` 根据闭环上下文展示当前标题与默认关键词

### 阶段 9-10：答辩与最终打包

已完成内容：

- 演示脚本
- 论文材料
- README 和文档站统一
- 世界线项目说明完善

## 当前项目状态

当前项目处于“毕业设计可交付版”阶段。

它已经不是一个“半成品想法”，而是一个：

- 能运行
- 能演示
- 能讲清楚
- 有明确后续迭代方向

## 后续建议阶段

### 阶段 11：展示层清理

优先级最高，建议尽快完成：

- 清理前端乱码
- 清理零散旧文案
- 统一展示层中文与品牌语义

### 阶段 12：PoE 数据正式接线

建议内容：

- 把样本知识卡接入知识库导入流程
- 把推荐候选接入更正式的规则执行层
- 把图谱样本接入图谱展示查询层

### 阶段 13：答辩前回归与截图固化

建议内容：

- 全链路回归
- 固定截图素材
- 固定答辩展示账号与演示脚本
