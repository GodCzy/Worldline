# 世界线 1.0 测试与验收手册

## 1. 文档目的

这份文档用于帮助项目作者、答辩老师和测试人员在本地完整验证“世界线”1.0 版本。

本文档回答四个问题：

1. 当前 1.0 版本已经做到什么程度
2. 本地启动后应该先看什么
3. 每个核心页面和功能该怎么测
4. 发现问题后如何判断是配置问题、使用问题还是系统缺陷

## 2. 1.0 版本当前能力范围

“世界线”当前已经具备毕业设计可交付级别的基础能力：

- 支持本地完整运行前端、后端、数据库、图数据库、向量库、任务 worker
- 保留 Yuxi-Know 的知识库管理、知识图谱、智能体、后台配置等平台底座能力
- 已完成世界线平台层改造：
  - 品牌替换
  - 首页与主题分区
  - 主题上下文透传
  - PoE 主题入口
- 已完成 PoE 首期样本模块：
  - 主题页
  - 推荐候选
  - 图谱闭环样本
  - Agent 与 Graph 的平台层联动
- 已完成答辩所需文档：
  - 项目说明
  - 架构说明
  - PoE 知识设计
  - 演示脚本
  - 论文材料
  - 完整使用教程

## 3. 当前 1.0 已知边界

1.0 版本已经可演示、可答辩，但不等于所有能力都已经做成最终产品态。

当前仍属于“可交付毕业设计版本”的部分包括：

- PoE 知识目前以样本资产和演示链路为主，还不是大规模正式知识库
- 推荐功能目前是 MVP 级别，重点在闭环展示和上下文联动
- 图谱闭环目前是样本展示型数据，不是完整线上图数据库建模
- Agent 主链路已经可进入、可承接主题上下文，但真实模型响应仍依赖可用 API 额度

## 6. 核心功能逐项测试

### 6.1 健康检查

访问：

```text
http://localhost:5050/api/system/health
```

通过标准：

- 返回 `200`
- 能看到健康状态 JSON

失败排查：

- 若页面打不开，先执行 `docker compose ps`
- 若 `api` 非 healthy，执行：

```powershell
docker compose logs api --tail=200
```

### 6.2 首页测试

访问：

```text
http://localhost:5173/
```

检查点：

- 浏览器标题是“世界线 - Worldline”
- 首页品牌是 Worldline / 世界线
- 页面能看到“主题分区”入口
- 顶部或侧边导航能进入主题页、智能体、图谱、知识库、Dashboard

通过标准：

- 成功进入系统内部页面

### 6.6 Agent 页面测试

访问：

```text
http://localhost:5173/agent/ChatbotAgent
```

如果你是从 PoE 主题页进入，则要重点检查上下文联动。

#### 页面结构理解

Agent 页面分为 4 块：

1. 顶部主题上下文区
2. 推荐 / 图谱说明区
3. 聊天记录区
4. 页面最底部的真实输入区

注意：

- 上方的提示卡片不是输入框
- 真正输入问题的位置永远在页面底部
- 现在页面上已经加入了引导文案，明确告诉用户这一点

#### 功能测试项

1. 新建对话
2. 选择历史对话
3. 输入消息
4. 发送消息
5. 查看右上角更多菜单
6. 查看推荐上下文与图谱上下文是否显示
7. 收起 / 展开推荐欢迎提示

#### 通过标准

- 首屏能看到输入框，或者非常容易看到输入区
- 输入框可输入、可发送
- 右上角“三个点”能点开
- 推荐上下文卡和闭环卡显示正常
- “收起提示”后页面不会把输入区挤走

#### 当前特殊说明

如果消息发送后返回模型权限或余额错误，不代表 Agent 页坏了，而是说明：

- 前端路由正常
- 对话链路已启动
- 但外部模型服务额度不足

这属于第三方模型资源问题，不属于页面结构 bug。

## 8. 常见问题判断表

### 8.1 页面打不开

优先检查：

```powershell
docker compose ps
docker compose logs web --tail=100
docker compose logs api --tail=100
```

### 8.2 登录失败

优先检查：

- 用户名密码是否正确
- `api` 是否 healthy
- 浏览器是否缓存旧页面

### 8.3 Agent 页“看起来没有输入框”

判断方式：

- 看页面最底部是否有输入区
- 看上方是否只是上下文说明区
- 尝试点“收起提示”

若输入区仍然不可见，再记录为布局 bug。

### 8.4 图谱页能开但查不到内容

这通常不等于页面坏了，可能是：

- 当前只是展示样本闭环
- 图数据库中没有对应导入内容
- 当前查询关键词不合适

### 8.5 能进聊天页但模型不回复

优先区分两种情况：

1. 页面没反应、输入区没法用  
这属于前端页面问题

2. 请求发出去了，但返回余额不足 / 权限不足  
这属于第三方模型服务问题
