# 09 企业化演进路线图

## 1. 文档目的

本文档用于定义 Worldline 的长期演进路线，确保后续每个阶段都围绕同一条产品主线推进。

Worldline 的长期方向不再只是“企业知识平台”，而是：

> **知识世界线引擎。**

它的目标不是只给出一个答案，而是让用户：

- 提出问题
- 看到多条可行世界线
- 选择并推进其中一条
- 在变化中切换、校正和再生成
- 最终基于知识、图谱、文档和模块能力构建自己的世界线系统

## 2. 当前产品基线

当前仓库已经具备：

- 平台基础：
  - 品牌、导航、模块入口、后台、文档、版本冻结机制
- 运行基础：
  - 基础聊天
  - Graph
  - Database
  - Dashboard
- 模块样板：
  - PoE
  - 推荐候选
  - 图谱闭环
  - 模块资产
- 工程基础：
  - MCP-first
  - subagent-first
  - 版本规划与缓存沉淀

## 3. 产品长期主线

Worldline 的长期主线不是“把知识堆得更多”，而是：

1. 让知识从静态资料变成世界线
2. 让世界线从解释工具变成推进工具
3. 让模块从演示样板变成可复制模板
4. 让用户最终能够基于知识资产创建自己的模块

## 4. 阶段路线

### v1.1 稳定化与治理化版本

已完成。

核心成果：

- 平台链路稳定
- 页面与主链路基线冻结
- 本地聊天运行时切换到 Ark
- `v1.1` 已冻结

### v1.2 顶层世界线工作区第一版

这是当前阶段。

核心任务：

- 保留基础聊天界面
- 新增主壳层顶层 `世界线` 入口
- 在世界线工作区内选择模块
- 由模型生成基础世界线分支
- 用简洁高端的分支舞台展示世界线
- 让节点可查看知识依据并继续生成下一步
- 打通到 Agent / Graph / 文档的承接

### v1.3 世界线增强版

重点方向：

- 世界线比较
- 世界线切换
- 世界线校正
- 更强的证据层
- 更强的模块语法差异

### v1.4 知识导入与模块生成增强版

重点方向：

- 导入知识库 / 图谱 / 文档
- 用导入资产增强世界线
- 初步形成自定义模块骨架

### v2.0 世界线模块平台

长期目标：

- 多模块共存
- 模块语法可定义
- 用户可生成和管理自己的模块
- 更强的治理、审核、发布和运维能力

## 5. MCP-first 路线

MCP 是 Worldline 的核心能力线之一。

当前第一批核心 MCP：

- GitHub
- PostgreSQL
- Fetch
- Playwright

优先服务：

- 版本规划与发布治理
- 数据与状态检查
- 外部资料提取与证据增强
- 浏览器级验收与回归

后续新增 MCP 必须满足一个原则：

> **能明显减少当前阶段的真实摩擦，而不是为了类别完整性。**

## 6. 当前阶段不纳入主线的事项

以下方向重要，但不应混入当前主线：

- 全量知识导入升级
- 完整 RAG 深化
- 图谱生产化编辑器
- 多智能体正式编排
- 大规模推荐引擎升级
- 模型栈一次性全量迁移

## 7. 主控结论

Worldline 现在最重要的，不是证明“我也有知识库 / 图谱 / Agent”，而是：

> **把“世界线”真正做成平台级工作方式。**

因此，`v1.2` 的真正任务不是继续堆功能，而是建立顶层世界线工作区，让用户第一次真正体验到：

- 先看多条未来路径
- 再选一条线
- 再沿线推进

这才是 Worldline 走向独特赛道的起点。
