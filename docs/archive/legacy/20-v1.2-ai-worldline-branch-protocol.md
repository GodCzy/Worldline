# v1.2 AI 基础世界线分支协议

## 定位

这份协议只服务 `v1.2` 第一版的世界线工作区。

目标不是直接实现完整动态世界线引擎，而是先冻结一套最小输入输出边界，让前端工作区、后续模型生成和知识支撑都围绕同一份结构工作。

第一版只覆盖四件事：

1. 用户输入问题，生成一组基础世界线分支
2. 用户点击某个节点，查看这条线的节点详情
3. 用户沿当前主线继续生成下一层世界线
4. 节点详情可以承接知识库、知识图谱和文档证据

## 总体原则

### 原则 1：前端不发明世界线语义

前端负责：

- 工作区布局
- 节点视觉
- 选中态
- 路径焦点
- 交互流

前端不负责：

- 发明分支含义
- 自己拼接“为什么是这条线”
- 自己发明风险、代价和推荐理由

这些语义字段应该来自模型或后续检索层。

### 原则 2：基础分支与后续分支尽量同构

“继续生成”不是另一套结构，而是沿当前主线继续返回同样结构的分支，只是多了父节点和深度信息。

### 原则 3：节点详情按需加载

节点详情不应该首屏全量展开，而是用户点击某个分支后再请求。

### 原则 4：证据对象与展示文本分离

证据对象身份由系统或检索层提供，解释文本可由模型生成。

## 1. 问题输入协议

### `WorldlineQuestionInputV1`

```json
{
  "protocol_version": "worldline.v1",
  "generation_mode": "base",
  "theme_id": "poe",
  "module_id": "poe",
  "user_question": "如果我是第一次接触 PoE，我应该从哪条世界线开始？",
  "goal": "找到最适合新手的起步路径",
  "constraints": {
    "budget": "low",
    "time_budget": "medium",
    "experience_level": "beginner",
    "risk_preference": "low"
  },
  "preferences": {
    "playstyle": ["minion", "safe"],
    "pace": "stable"
  },
  "knowledge_scope": {
    "use_knowledge_base": true,
    "use_graph": true,
    "use_docs": true
  },
  "context_refs": {
    "theme_context": {
      "theme": "poe",
      "module": "poe"
    },
```

- `protocol_version`

- `context_refs`

### 应由模型生成

- `question_digest`
- `workspace_title`
- `workspace_subtitle`
- `branch.title`
- `branch.subtitle`
- `branch.summary`
- `branch.why_this_line`
- `branch.fit_for`
- `branch.risk_level`
- `branch.cost_level`
- `branch.complexity_level`
- `branch.confidence_score`
- `branch.preview_next_question`
- `branch.continuation_hint`
- `detail_title`
- `detail_summary`
- `detail_sections`
- `suggested_next_generation_prompt`

### 应由系统或检索层提供

- `ref_id`
- `ref_type`
- `title`

### 应由前端补充

- 节点坐标
- 连线样式
- 视觉层级
- 选中态
- 面板开合状态
- 动画和主题皮肤

## 6. 第一版边界

### 第一版必须有

- 一次问题输入
- 一次基础分支生成
- 点击节点查看详情
- 沿当前线继续生成下一层
- 节点详情承接知识库 / 图谱 / 文档依据
- 前端画布消费统一结构化分支

### 第一版不要做

- 无限递归生成
- 自动比较所有世界线
- 自动校正全部历史世界线
- 拖拽编辑
- 重型图编辑器
- 多智能体共生成整棵树
- 全量节点详情预取

### 建议限制

- 基础分支数：`2 ~ 4`
- 每次继续生成新增分支：`2 ~ 3`
- 最大深度：`2` 或 `3`
- 节点详情：按需请求

## 7. 与当前前端骨架的映射

- `WorldlineQuestionBar`
  - 提交 `WorldlineQuestionInputV1`
- `worldlineContext`
  - 保存 `root_question / branches / active_branch_id / selected_node_id / generation_depth`
- `poeWorldlineAdapter`
  - 当前先作为协议兼容层，模拟模型输出
- `WorldlineBranchCanvas`
  - 只消费统一化后的 `branches + parent_node_id + depth`
- `WorldlineBranchDetailPanel`
  - 对应 `WorldlineNodeDetailResponseV1`
- `WorldlineNextStepPanel`
  - 触发 `WorldlineContinuationRequestV1`

## 当前结论

`v1.2` 第一版不需要完整动态世界线引擎，但必须从现在开始使用统一协议思维。

否则后续一旦接入真实模型生成，前端会重新退回“静态推荐页 + 前端假树”的旧问题。
