# phase-39 v1.2 世界线工作区纠偏

## 本轮稳定结论

`v1.2` 的核心方向已经从“独立 PoE 工作台页”正式纠偏为：

> **主壳层中的顶层世界线工作区。**

这次纠偏解决了 5 个关键误差：

1. 世界线不应再被当成某个主题页里的独立功能页。
2. 世界线应成为主壳层左侧导航中的一级入口。
3. 进入世界线后，应先选择模块，再进入模块工作区。
4. 世界线分支不应再主要依赖静态写死文案，而应由模型根据问题生成基础分支，并在用户选择后继续生成后续分支。
5. 工作区不应像推荐页或聊天页附属边栏，而应像一个简洁、高端、独立的路径舞台。

## 当前正确的产品结构

- 基础聊天界面继续保留
- 新增顶层 `世界线` 工作区
- 世界线工作区中先做模块选择
- 模块工作区中进行：
  - 问题输入
  - 模型生成基础世界线
  - 选择世界线
  - 节点详情查看
  - 证据查看
  - 下一步生成
  - Agent / Graph / 文档承接

## 路由与入口方向

推荐路由：

- `/worldline`
- `/worldline/:themeId`

现有原型路由：

- `/themes/:themeId/worldline`

后续应降级为：

- 模块启动入口
- 兼容跳转入口

## 第一版实现红线

第一版绝不能退化成：

- 推荐卡片页
- 聊天页附属边栏
- 后台风格配置页

第一版也绝不能提前混入：

- 重型动态世界线引擎
- 正式比较器 / 切换器 / 校正器
- 拖拽编辑器
- 全量知识导入升级
- 多智能体正式编排

## 第一版最小成功标准

1. 用户从左侧导航进入 `世界线`
2. 首屏先看到工作区主舞台，而不是聊天流
3. 用户能看到 `2 ~ 4` 条结构化基础世界线
4. 选中某条线后能看到证据与下一步
5. 用户能沿当前线继续进入 Agent / Graph / 文档
6. 用户能清晰感知：
   - 这是新的工作模式
   - 不是普通聊天页增强版

## 阶段判断

- 当前阶段：`v1.2 规划`
- readiness：`ready 进入 v1.2 实施准备下一轮`
- 当前最大 gap：需要把顶层工作区进一步落实为文件级实现基线，并在下一轮开始真正前端实现
