# phase-40-v1.2-worldline-shell-pass1

## 本轮目标

把“世界线”从 PoE 主题内的独立原型页，提升为主壳层里的顶层工作区入口。

## 稳定结论

- `世界线` 已进入主壳层左侧导航，成为与首页、主题分区、智能体并列的顶层入口。
- 新增顶层路由：
  - `/worldline`
  - `/worldline/:themeId`
- 旧入口 `/themes/:themeId/worldline` 已降级为兼容跳转入口，避免旧链接失效。
- 新增 `WorldlineHubView.vue`，负责先选模块，再进入模块内世界线工作区。
- `WorldlineWorkbenchView.vue` 已从“PoE 独立原型页”转为模块工作区页，支持从顶层世界线入口进入。
- `ThemeDetailView.vue` 中原“进入世界线工作台”入口已改为模块启动入口，并指向新的顶层世界线路由。

## 本轮边界

- 只改前端壳层、导航、路由、入口和低风险页面文案。
- 不改聊天核心链路。
- 不改知识库、图谱、RAG、Agent 核心运行逻辑。
- 不做动态世界线生成器。
- 不做重型图编辑器或复杂自动布局。

## 当前产品形态

当前正确形态已经从：

- 主题页里的一个世界线原型入口

推进为：

- 主壳层顶层 `世界线`
- 进入后先看模块总览
- 再进入某个模块的世界线工作区

## 已验证事实

- 前端构建通过：`pnpm build`
- 新路由可达：
  - `/worldline`
  - `/worldline/poe`
  - `/themes/poe/worldline`
  - `/themes/poe`

## 仍未完成

- 还没有把“模型生成基础分支”真正接入工作台。
- 当前工作台仍主要依赖静态 adapter 数据。
- 视觉语言还没有达到最终目标的“简约、高端、震撼”。
- 不同模块的差异化 UI 仍未开始。

## 阶段判断

- 当前阶段：`v1.2 实施`
- readiness：`ready`
- 下一步最主要 gap：让世界线工作区真正出现“AI 生成基础分支”的体验，而不只是顶层入口和静态分支壳层。
