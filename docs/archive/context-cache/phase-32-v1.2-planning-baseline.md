# Phase 32 - v1.2 第一版规划基线

## 稳定结论

- `v1.1` 已冻结
- `v1.2` 已进入规划阶段
- `v1.2` 第一版定位为：`企业化运营与工程效率增强版`
- `v1.2` 第一版不做大功能扩张

## MCP-first 重点

`v1.2` 最先服务的三类高频工作：

1. 仓库协作与发布流转
2. 数据与状态检查
3. 页面验收与回归

支撑型 MCP：

- `Fetch` 用于外部资料拉取与知识素材整理

当前判断：

- 第一批 MCP 已够用
- 暂不需要新增更多 MCP

## 模型栈治理结论

- `v1.2` 第一版会纳入模型栈治理
- 但只纳入治理基线，不纳入全量迁移执行
- 聊天、Embedding、Reranker、内容安全模型的统一迁移应后续分步推进

## 明确延后事项

以下内容不纳入 `v1.2` 第一版：

- PoE 正式知识导入批量落地
- RAG 检索优化
- 图谱生产化
- 推荐系统升级
- 多智能体正式编排
- 系统设置深治理
- 全站视觉重构

## 阶段判断

- 当前阶段：`v1.2 规划`
- 下一阶段 readiness：`ready`
- 下一阶段：`v1.2 实施准备`
- 进入下一阶段前的主要 gap：
  - 需要把 MCP-first 的项目级落地方式写清楚
  - 需要把 `v1.2` 的验收门槛与实施顺序正式冻结
