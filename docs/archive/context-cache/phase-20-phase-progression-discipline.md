# 世界线 Phase 20 阶段推进纪律

## Goal

- 让项目后续完善按明确阶段推进，而不是零散任务堆叠。
- 固化新的执行纪律：每轮 substantial task 结束后，主控都要主动判断当前阶段、下一阶段 readiness 和主要缺口。

## Stable Decisions

- 世界线后续必须按阶段推进。
- 主控代理不能只完成当前任务，还要主动回答三个问题：
  - 当前处于哪个阶段
  - 离下一阶段还差什么
  - 是否已经可以进入下一阶段
- 这项判断不应等待用户提醒。
- 如果阶段 readiness 发生变化，应写入 `docs/context-cache/`。
