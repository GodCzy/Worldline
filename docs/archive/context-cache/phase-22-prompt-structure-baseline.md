# 世界线 Phase 22：提示词结构基线

## Goal

- 固定后续所有建议提示词的输出结构
- 让用户可以直接复制给 Codex，减少来回改写成本

## Stable Decisions

- 从本阶段开始，每次 substantial task 结束后，控制器都必须给出且只给出一个建议提示词
- 该提示词必须使用以下 5 段结构：
  - `先读：`
  - `当前基线：`
  - `本轮优先任务：`
  - `主控最后汇总：`
  - `要求：`
- `本轮优先任务：` 是提示词中最重要的部分，必须写清楚 subagent 的任务分配，避免浪费常驻 subagent 阵型
- `主控最后汇总：` 必须明确写出主控在 subagents 完成后需要：
  - 汇总什么
  - 判断什么
  - 输出什么
- 如果任务是 substantial task，则提示词里必须显式体现：
  - subagent 分工
  - 验证要求
  - git 提交要求
  - `docs/context-cache/` 更新要求
