# Context Cache

这个目录用于存放可被 Codex 快速读取的压缩上下文文档。

用途：

- 在上下文不足时，优先读取这里的摘要，而不是重新扫描全仓库。
- 为多 agents 协作提供统一基线，减少重复分析。
- 固化每个阶段的目标、边界、风险和下一步动作。

当前建议优先阅读顺序：

1. `phase-1-baseline.md`

维护规则：

- 只保留高密度、可执行、与当前阶段直接相关的信息。
- 阶段切换时追加新文件，不覆盖已完成阶段的基线记录。
- 内容以项目边界、结构映射、实施顺序、风险提示为主，不写冗长过程性说明。
- Codex 在任务结束前应主动检查这里是否需要新增或更新摘要，不需要等待用户提醒。
