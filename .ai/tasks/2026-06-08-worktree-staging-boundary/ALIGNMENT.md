# P2-4 工作树分组与提交边界

日期：2026-06-08

## 目标

对当前 `codex/worldline-recovery-refactor` 的脏工作树做分组审计，形成后续提交边界，避免把旧阶段删除、历史任务证据、前端 QA、后端合约和临时清理证据混在一次提交里。

## 边界

- 本任务只读 git 状态并写审计文档。
- 不执行 `git add`、`git commit`、`git reset` 或回滚。
- 不判断所有旧修改是否都应该进入最终 PR，只给出分组和复核顺序。
- 不覆盖用户或历史 Agent 的修改。

## 验收

- 记录当前 `git status` 规模和主要路径分布。
- 把已跟踪修改、删除和未跟踪文件分开。
- 给出推荐提交包与不建议直接提交的路径。
- 写入 `.ai/tasks` 证据和 `D:\document\OutputMD` 总结。
